---
name: read-paper
description: Read a PDF and auto-generate a structured paper note. Output defaults to `unreviewed` and must pass `/review-note` before being citable by the wiki.
---

Read the PDF I specify, then do the following:

## Step 1: Create the paper note

Create a new paper note at the vault root, with this structure:

1. YAML frontmatter — **must** include the `review_status` field:
   ```yaml
   ---
   title: "Full paper title"
   authors: [author list]
   year: YYYY
   venue: "Full journal / conference name"
   tags: [paper, ...other relevant tags]
   status: reading
   review_status: unreviewed        # REQUIRED: marks this as auto-generated and not yet audited
   reviewed_date: null
   reviewed_by: null
   review_notes: null
   ---
   ```

2. A prominent review-status banner at the very top of the note body:
   ```markdown
   > [UNREVIEWED] This note was auto-generated by `/read-paper` and has not been audited. Content may be wrong (summarization errors, wrong numbers, misread concepts). It must pass `/review-note <note-name>` before the wiki is allowed to cite it.
   ```

3. Paper info (title, authors, year, keywords)
4. Research question — what problem is the paper trying to solve?
5. System model / scenario assumptions — network architecture, communication model, etc.
6. Proposed method — core algorithm / framework
7. Key technical details — formulas, RL reward design, optimization objective
8. Experimental setup — simulator, parameters, baselines
9. Main results — key numbers and findings
10. Strengths and weaknesses — be specific, not generic
11. Relation to my research — connect to the user's domain defined in SKILL.md
12. Key notes / reactions — leave blank for the user to fill in later
13. Related concepts — links

## Step 2: Auto-link to existing notes

Scan every existing note in the vault and find ones related to this paper:
- Papers using the same method or technique
- Papers addressing the same problem
- Concept notes in the user's domain

In the new note's "Related concepts" section, list all matches as `[[wiki-links]]`.
Also open each related existing note and add a reverse link back to the new note.

Because the new note is `unreviewed`, when adding a reverse link in an existing note, append an `[UNREVIEWED]` marker next to the link:
```markdown
[[new-paper-note]] [UNREVIEWED]
```
The marker can be removed automatically by `/review-note`'s cascade step once the note is reviewed.

## Step 3: Extract and create concept notes

Extract 3–5 core concepts from the paper:
- If a matching concept note already exists in the vault, add a link to this paper inside it (also tagged `[UNREVIEWED]`).
- If no matching note exists, create one with a definition and a link to this paper.
- Place concept notes at the vault root.

## Step 4: Prompt for review

When finished, show a prominent reminder:

```
[DONE] Note created: [[new-paper-note]]
[UNREVIEWED] State is unreviewed. Run `/review-note <note-name>` next to verify the note against the PDF, otherwise the wiki cannot cite it.
```

## Requirements

- Use the output language specified in SKILL.md's Researcher profile.
- Keep technical terms in their original form.
- File name: short English slug derived from the paper's title.
- `tags` must include `paper` plus relevant domain keywords.
- `status` is `reading`.
- `review_status` **must** be `unreviewed` — never default to `reviewed`.
- Final report: list which notes were created and which were modified, and explicitly flag the `unreviewed` state.
